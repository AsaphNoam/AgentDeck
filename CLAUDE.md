# AgentDeck ‚Äî agent context

Multi-agent dashboard. Go server (module `github.com/agentdeck/agentdeck`, Go 1.25 ‚Äî cobra CLI,
SQLite via `mattn/go-sqlite3` with FTS5 behind the `sqlite_fts5` build tag, `creack/pty`, ACP over
`modelcontextprotocol/go-sdk`) with an embedded React 18 + Vite + TypeScript UI in `ui/`
(Zustand, React Query, Radix, `@xterm/xterm`; UI tests: Vitest + Testing Library + MSW).

## Build & verify ‚Äî the GREEN checkpoint
A change is done only when ALL of these pass (never commit on red):

```sh
go build ./...
go test ./...
go test -tags sqlite_fts5 ./...
cd ui && npm run build        # only if the change touched ui/
```

- Both test variants matter: the untagged run covers the shipped **no-FTS5 fallback path**. A fix
  that only passes under `-tags sqlite_fts5` is not green, and vice versa.
- The `sqlite_fts5` tag must be on every **real build path** (`make build`, `install.sh`) ‚Äî a
  missing tag once shipped a binary whose `MATCH` queries fail at runtime. When touching build
  scripts, verify the *build* command line, not just the test one.
- `make test` runs both variants; `make dist` = `make embed` (copies `ui/dist` ‚Üí
  `internal/server/ui/dist`) + tagged `go build`.
- **Never edit `internal/server/ui/dist/**`** ‚Äî it is generated by `make embed`. Edit `ui/src` and
  rebuild. (A hook blocks this.)
- `make test` does **not** use `-race`, and every race in the review history was found by human
  reasoning, not tooling. When you touch `internal/runtime`, `internal/bus`, or anything with
  goroutines/locks, also run `go test -race` on those packages before calling it green.
- UI dev loop: `cd ui && npm run dev` (port 5173, proxies `/api` ‚Üí `:4317`); dev-mode server:
  `go run -tags dev ./cmd/agentdeck dashboard start`. UI unit tests: `cd ui && npm run test`.
- `.gocache/` at the repo root is a gitignored local Go build cache ‚Äî ignore it; never `git add` it.

## Commits & history
- Trunk-based: commit directly to `main`; no branches, no PRs. **Never push to origin unless the
  human explicitly asked** (a hook will prompt if you try).
- Message templates: `phase N.M: <title> ‚Äî green checkpoint` (build loop), `review fix: <title> ‚Äî
  green checkpoint` (fix loop); plain `docs:`/`feat:` prefixes for out-of-loop commits. Add a
  `Co-Authored-By:` trailer naming the model that actually did the work.
- Mining history: a past rehash-and-merge left many **tree-identical duplicate** commit pairs ‚Äî
  dedupe by commit subject, not SHA.
- Test conventions: one `_test.go` per source file; regression tests are named after the defect they
  guard (`TestCrashTearsDownAgentRegistration`, `TestPathTraversalRejected`). Protocol-level repros
  go through the env-driven fake ACP peer `internal/runtime/testdata/fakeacp` (built once via
  `buildFakeACP` in `internal/server/integration_test.go`).

## Hot spots ‚Äî where review findings concentrated
- **`internal/server/`** (`launch.go`, `resume.go`, `switch.go`, `sessions.go`, `reconcile.go`) ‚Äî
  the seam where config, runtime, state, hooks, and MCP registration compose. Launch, resume, and
  switch each rebuild a LaunchSpec: any field or cleanup step added to one path gets forgotten in
  the others. Always go through the shared helpers ‚Äî `composeLaunch`, `resolveSkipForRole`,
  `resolveAddDirs`, `teardownAgentRegistration` ‚Äî never hand-roll a subset.
- **`internal/runtime/`** (`chat.go`, `permission.go`, `terminal/`) ‚Äî process lifecycle and
  permission/cancel races. The atomic-claim (`takePending`) and turn-generation (`turnSeq`)
  patterns live here; reuse them for any new check-then-act transition.
- **`internal/state/` + `internal/index/`** ‚Äî in-memory accumulators feeding durable tables across
  restarts (`seedLocked` pattern), build-tag capability gaps, `rows.Err()` discipline.
- **`ui/src/`** ‚Äî SSE reconnect state resets, forms over seeded server config (merge, never
  replace), and mutating actions that must surface errors (`.catch ‚Üí pushError`, no bare `void`).
