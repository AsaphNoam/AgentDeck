# AgentDeck — Claude Code context

AgentDeck is a local Go server with an embedded React/Vite UI that supervises agent CLIs. This file
contains Claude Code adapter guidance only; product and architecture rules live in `docs/specs/`.

## Start every role

Read, in order:

1. `docs/features/HANDOFF.md`
2. `docs/specs/README.md` and the governing FS/TS/INV items named by the handoff/request
3. `docs/features/AGENT-WORKFLOW.md`

`AGENTS.md` routes other agents to the same contracts. Archived phase docs, `MAP.md`,
`architecture-flow.md`, ADRs, plans, reviews, and briefs are non-authoritative.

## Claude tooling

- `.claude/hooks/session-start.sh` injects only the handoff header; still read the full handoff.
- Edit/Write hooks guard generated UI/cache paths, formatting, recurring invariants, twinned skills,
  and local spec mechanics. Hook feedback does not replace the GREEN checkpoint.
- `.claude/skills/*` and `.agents/skills/*` are byte-identical launchers into the canonical workflow.
  Update both copies together.
- Use `ui/src` as the UI source. `internal/server/ui/dist/**` is generated by `make embed`.
- `.gocache/` is a gitignored local cache; never stage it.

## Verification and history

TS-06 and workflow §2 define the shared commands and proportional docs-only checkpoint. Both Go
test variants matter; concurrency hot spots add focused `-race` coverage. Trunk/commit/brief rules
live only in the workflow. A historical rehash left tree-identical duplicate commits, so deduplicate
history by subject as well as SHA.

Answer directly and keep the final response byte-for-byte identical to the stored session brief.
